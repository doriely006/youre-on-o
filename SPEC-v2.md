# You're On — v2 Redesign Spec

> v1 是「追蹤器加上評論」，v2 是「節目加上追蹤」。
> 數字只是劇本，主角是這場秀。

## 設計原則

- 核心樂趣一句話：填時數，被觀眾嘴。所有使用者「必須做」的事都是摩擦
- 目標用戶是容易棄坑的人 → 最壞情境（落後很多）必須設計成想回來，而不是想刪 app
- 用敘事壓力取代數學債務，用社交成本取代機制成本
- 三層迴路各管一件事：
  - **日 = 行為**（有沒有出現）→ 靠嘴
  - **月 = 方向**（時間分配像不像你說的那個人）→ 靠儀式
  - **宣言 = 決心**（為什麼開這個節目）→ 靠錨點

## 與 v1 的差異總表

| | v1（現況） | v2 |
|---|---|---|
| 核心數字 | 達成率 ×3 維度 | 觀眾數 ×1 |
| 落後的代價 | 時數債務累積（gaps） | 觀眾流失 |
| 改目標 | 72h 冷靜期 | 隨時可改，但觀眾永久記得並翻舊帳 |
| 月維度 | 第三個進度環 | 月底改版會議（儀式，非監控） |
| 評論 | 無狀態隨機抽取 | 有記憶的角色 + 事件 + 錯位評論 |
| 缺口還款方式 | 三選一設定 | 移除 |
| 簡單/進階模式 | 兩種模式 | 移除（v2 本身就夠簡單） |
| 最壞情境 | 缺口 23h → 刪 app | 回歸特輯 → 最爽的一集 |

## 1. 觀眾數（核心數字）

- 取代達成率矩陣成為 dashboard 第一眼看到的數字：「📺 目前收看人數：1,847（↓ 23）」
- 機制（初版，數值可調）：
  - 起始觀眾：100
  - 每日有記錄：+觀眾（基礎 +5，當日達標加成至 +15）
  - 每日無記錄（停播）：-觀眾（遞增：第 1 天 -3、第 2 天 -8、第 3 天起每天 -15）
  - 週 100% 達成：爆量 +50
  - 觀眾數下限 1（最後一個觀眾是阿姨，她不會走，但她會講話）
- 觀眾數變化是評論素材：「觀眾跌破 500，製作單位開始討論時段調整」
- 達成率仍然計算（評論和總表需要），但降為第二層資訊

## 2. 播出機制（記錄 = 集數）

- 每日記錄 = 「今日播出」；記錄頁送出按鈕改為「播出本集」
- 不記錄 = 停播；連續記錄 = 連播 N 集（顯示在 dashboard）
- 停播 3 天以上後回來記錄 = **回歸特輯**：
  - 特殊評論（觀眾好奇你變怎樣了）
  - 觀眾數一次性回補（流失量的 50%）
  - 這是全 app 設計上最重要的時刻：回歸必須是爽的

## 3. 開場宣言

- Setup 多一個必填欄位：「你為什麼開這個節目？」（一句話，例：年底前轉職成功）
- 宣言顯示在銀幕上方，是節目的前提
- 宣言是評論素材：觀眾會引用你自己的話打你的臉（含設定日期）
- 宣言可改，但舊宣言保留在歷史，觀眾記得

## 4. 評論系統 v2

### 4a. 角色記憶
- 五個常駐角色（朋友/同事/老闆/阿姨/前任）各有人設和出現偏好：
  - 前任：專挑你連續達標時出現潑冷水
  - 阿姨：停播越久越關心你
  - 老闆：只看數字
- 角色會引用上次互動：「你上週也是這樣說的」（儲存 per-character 最後互動摘要）

### 4b. 錯位評論（新類型，最高優先）
- 素材不是達成率，是「宣言/目標比例」vs「實際時間分配」的落差
- 範例語氣：「你設定的人生是 50% 找工作，這個月實際 23%。你不是做不到，你是其實不想，差別很大喔」
- 觸發：實際累計比例與目標比例偏差 >15 個百分點的 category

### 4c. 事件評論
- 里程碑：累積第 100 小時、首次週 100%、連播 7 集、觀眾破千
- 負面事件：目標修改（當場評論 + 之後隨機翻舊帳「他第 N 次調降目標了」）、觀眾數跌破關卡
- 回歸特輯（見 §2）

### 4d. 內容量
- 三種格式保留（PTT 50% / LINE 40% / 財經 10%）
- 每 tier 每格式至少 15 條（總量 300+），由 Claude 產生初稿、使用者挑選修改
- 模板變數擴充：{cat} {pct} {hrs} {dim} {aud}觀眾數 {streak}連播 {declaration}宣言 {edits}改目標次數 {days}停播天數

## 5. 月底改版會議（儀式）

- 每月最後一天（或次月首次打開 app）強制觸發，全螢幕，不可跳過
- 新角色「製作人」主持：全 app 唯一不嘴你、只問問題的角色
- 攤開本月總表（實際累計比例 vs 目標比例），逼三選一：
  - **a. 調降/調整目標** — 公開事件，觀眾評論分裂（有人嘴慫，有人說務實）
  - **b. 維持目標** — 觀眾：「好，我們看著你」；下月錯位評論加倍毒
  - **c. 方向沒錯、執行有問題** — 進入決心檢驗：下月觀眾只評「說到做到」
- 選擇被記錄，成為下月評論素材
- 改目標不是罪，無聲漂移才是：月會上公開調整 = 被尊重；會外偷改 = 被嘴爆

## 6. 砍掉的東西（v1 → v2 移除清單)

- 72h 冷靜期 + pendingGoals（改為自由修改 + 觀眾記憶）
- 缺口債務 gaps + 還款方式 repayMode
- 簡單/進階模式切換
- 本月進度環（月只活在改版會議）
- 週調整/月調整（weeklyAdjust/monthlyAdjust）

## 7. 保留的東西

- 三種評論格式與視覺（PTT 深藍論壇 / LINE 對話 / LIFE FINANCIAL TIMES）
- 影廳視覺：布幕、光暈、開場拉幕動畫、觀眾席
- category + 每週目標時數 + 自動換算比例
- 記錄頁全列表（v1 第一批的版本）
- 總表（實際累計比例 vs 目標比例）— 它現在是錯位評論和月會的資料來源
- 深色主題、Bebas Neue + Noto Sans TC、#FF2D20/#FF9F0A/#30D158

## 8. 技術

- 維持單檔 index.html（React CDN + babel standalone），maxWidth 430
- Storage key 升版：`reality_v6`，需寫 v5 → v6 migration（保留 categories/records，轉換或丟棄 v1 欄位）
- 新增資料欄位：audience、streak、lastAirDate、declaration{text,date,history[]}、charMemory{}、monthlyReviews[]、goalEditLog[]
- 加「匯出 JSON」按鈕（localStorage 是定時炸彈）
- GitHub Pages 自動部署（push to main 即上線）

## 9. 開發順序建議

1. 資料結構 v6 + migration + 觀眾數引擎 + 播出/停播/連播/回歸
2. 視覺系統重建（§10，design tokens + 基底元件先行，後續頁面直接用對的皮）
3. Dashboard 改版（觀眾數為主角）+ 宣言
4. 評論系統 v2（錯位評論 + 角色記憶 + 事件）
5. 月底改版會議
6. 評論內容量產（Claude 產初稿 → 使用者挑選修改）
7. 匯出 JSON + 砍 v1 廢棄功能

## 10. 視覺系統重做（方向：串流平台片頭感）

v1 視覺被判定 cheap，v2 全面重做。基調：Netflix/HBO 式深黑電影質感，
冷硬、有光、雜誌排版。「數字不會說謊」是冷的，視覺也要冷。

### 10a. v1 的廉價病因（全部修掉）
- Emoji 當 icon（📊✏️⚙️👁📺）→ 全部換成 inline SVG icon（線條風格，1.5px stroke，統一 24px 網格）
- CSS 條紋漸層假布幕 → 手繪 SVG 布幕：真實垂墜皺褶 path + 皺褶明暗 + 扇形下擺 + 受銀幕光影響的邊緣光
- 「卡片湯」（無限重複 #141414 圓角卡 + 1px 灰邊）→ 用排版層次取代卡片邊框，能不加框就不加框
- 原生 confirm()/alert() → 自製 modal（深黑 + 同視覺語言）
- 紅橘綠交通號誌三色 → 見 11c 配色紀律
- 沒有打光 → 見 11d 光系統

### 10b. 字體排印
- Bebas Neue 保留但有紀律：只用於「大數字」和「節目標題」兩種場合，禁止當一般 UI 標籤字體
- 大數字要夠大：觀眾數是全頁最大元素（72px+），達成率次之；雜誌邏輯 = 尺寸對比要狠，最大和最小差 5 倍以上
- Noto Sans TC 負責一切內文，用 weight（300/500/700）和透明度做層次，不靠顏色
- 字距：大標 letter-spacing 收緊，小標籤大寫 + 寬字距（tracking 3px+）做「片頭字卡」感

### 10c. 配色紀律
- 背景：比現在更黑的多層次黑（#050505 底 + 聚光漸層提亮局部），拒絕整面均勻灰黑
- **紅 #FF2D20 只留兩個用途：品牌（標題/ON AIR 燈）和負面事件。不再當按鈕主色**
- 輔色換成香檳金（#C9A86A 一帶）：達成、里程碑、觀眾數上升用金，取代現在的橘
- 綠只留一個用途：正向 delta（↑）。交通號誌三色並排的畫面不准出現
- 文字以白的透明度階層為主（100% / 60% / 35%），少用彩色文字

### 10d. 光系統（電影感的本質是打光）
- 銀幕：光暈保留但做成三層（內亮邊 / 中暈 / 遠距離 spill 打在觀眾席上）
- 全頁 vignette（四角暗角，radial-gradient 疊加）
- 膠片顆粒：SVG feTurbulence noise filter 疊全頁，透明度 3-4%，去掉數位平面感
- 聚光燈邏輯：重要元素（觀眾數、宣言）後方有微弱 radial 提亮，像被 spotlight 打到

### 10e. 元件與細節
- 銀幕區：加 ON AIR 紅燈（呼吸閃爍）+ 集數字卡（EP.47）— 方向 a 為主、保留少量播出元素
- 觀眾席剪影重畫：成排座椅椅背 + 後腦勺，至少兩排深度，前排被銀幕光照亮頂部輪廓
- 數字變化要動：觀眾數增減用 count-up/down 動畫 + 變化量浮出（+15 淡入上飄）
- 導航列：SVG icon + 極小寬字距大寫標籤（HOME / RECORD / SETTINGS），active 用金色不用紅色
- 所有過場 150-250ms ease-out，拒絕生硬切換；但動畫總量克制，cheap 的另一種形式是到處都在動

### 10f. 驗收標準
截圖丟給一個沒看過的人，他的第一反應應該是「這是什麼節目的 app」，
而不是「這是誰的 side project」。

## 11. 已知風險（誠實備註）

- 觀眾數機制的數值需要實測調整，初版數字是猜的
- 自我回報悖論無解也不用解（想騙觀眾打 4h 就好，誠實是前提）
- 達成率平均制可被小 category 灌水（嚴重度低，知道就好）
- 評論重複仍是長期風險，300+ 條只是及格線
